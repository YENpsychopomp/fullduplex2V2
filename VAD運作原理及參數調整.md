# VAD 運作原理及參數調整

## 運作流程與原理

VAD的運作可以拆解為以下三個層次：

### 1. 音訊數位化與標準化 (Preprocessing)
系統透過音訊輸入（如 PyAudio 或 WebSocket）以 16,000 Hz 的取樣頻率抓取聲音。Silero VAD 模型對輸入格式有嚴格要求：
- **取樣率**：必須剛好為 $16\text{ kHz}$。
- **長度**：每次處理固定 $512$ 個樣本（約 $32\text{ ms}$）。
- **振幅**：數據必須正規化至 $-1.0$ 到 $1.0$ 之間的浮點數（`float32`）。

### 2. 神經網路判斷
偵測階段分為兩層 AI 模型：
- **基礎語音偵測 (Silero VAD)**：
  每一塊 $32\text{ ms}$ 的聲音片段都會送入 ONNX 模型進行推論。模型會輸出一個介於 0.0 到 1.0 之間的機率值 ($P$)。在程式碼中，只要$P > VAD_{THRESHOLD}$即判定為正在講話。
- **語意完整性預測 (Smart Turn Detector)**：
  這是一套非同步（Async）執行的第二層模型推理 (`vad_inference.py`)。當使用者進入「停頓」時，系統會將這段時間累積的整段錄音送給這個模型，用來推估使用者「這句話是否講完了」。模型會回傳完整機率：
  - **$P \ge 0.70$ (`HIGH_CONFIDENCE`)**：表達極高機率已完整，系統立即斷句。
  - **$P \ge 0.50$ (`MEDIUM_CONFIDENCE`)**：可能講完，系統會稍微縮短觀察靜音的時間。
  - **$P < 0.50$ (`LOW_CONFIDENCE`)**：明顯沒講完，系統會拉長等待時間，給使用者思考。

### 3. 動態端點偵測 (Dynamic Endpoint Logic)
這是程式碼最核心也最聰明的部分。它並不是死板地看到靜音就立即斷句，而是維護一個狀態機來做智能判斷：
- **預熱緩衝 (Pre-buffer)**：隨時保留最近 $200\text{ ms}$ 的聲音資料。當使用者突然開口時，系統能把開頭微弱的氣音或母音也抓回來，避免開頭被截斷。
- **動態冷卻期**：當使用者講完話開始停頓時，系統會啟動 `EARLY_CHECK_MS`（例如 1.2 秒）。如果在停頓期間，模型回傳的「靜音置信度」非常高，系統會縮短等待時間（加速反應）；如果偵測到使用者可能還在呼吸或有細微雜音，系統則會延長等待時間（避免錯誤斷句）。

## 參數調整指南：針對不同場景

你可以根據實際的應用情境，調整程式碼開頭的這些全域變數：

### 場景 A：極速回應（如：語音遙控器、簡單指令）
如果你希望 AI 在使用者講完話的瞬間就立刻動作，且不在乎偶爾被提早斷句。
- `VAD_THRESHOLD`: **0.8** (提高觸發門檻，只抓取明確的聲音)
- `EARLY_CHECK_MS`: **500** (講完後只等待 0.5 秒便開始檢查)
- `MAX_STOP_MS`: **1000** (靜音最多等待 1 秒就強制斷句)

### 場景 B：自然對話（如：AI 老師、聊天機器人）
使用者可能會邊想邊說，需要給予他們一點「思考與停頓」的空間。
- `VAD_THRESHOLD`: **0.7** (標準值)
- `EARLY_CHECK_MS`: **1200** (給予使用者 1.2 秒的喘息空間)
- `MAX_STOP_MS`: **2500** (若背景稍有雜音，最多等待 2.5 秒)
- `MIN_CHECK_INTERVAL_MS`: **100** (讓系統頻繁確認是否真的已經講完)

### 場景 C：環境極度吵雜（如：工廠、戶外）
背景可能有強風、機器運轉聲等，VAD 容易發生誤判。
- `VAD_THRESHOLD`: **0.9** (條件非常嚴格，只有大聲講話才會觸發)
- `PRE_SPEECH_MS`: **300** (增加前置緩衝，確保不會漏掉被噪音蓋住的語音開頭)
- `HIGH_CONFIDENCE`: **0.85** (要求極高的靜音把握度才判定為斷句)

## 參數彙整表

| 參數名稱 | 建議範圍 | 影響說明 |
| :--- | :--- | :--- |
| **`VAD_THRESHOLD`** | $0.5 \sim 0.9$ | 語音靈敏度。數值越高越不容易被雜音觸發，但可能會漏掉微弱的氣音或語尾。 |
| **`EARLY_CHECK_MS`** | $500 \sim 2000$ | 第一反應時間。講完話後經過多少毫秒開始考慮切斷連線（進行端點偵測）。 |
| **`MAX_STOP_MS`** | $1000 \sim 5000$ | 系統的耐心極限。無論環境多吵，只要停頓超過這個時間，一定強制斷句。 |
| **`MAX_DURATION_SECONDS`** | $5 \sim 20$ | 強制中斷的總時長。防止使用者無止盡地一直講，強迫 AI 做出階段性回應。 |

> **調整小訣竅：**
> - 如果你發現 AI 常常在使用者話還沒講完（例如在句子中間換氣時）就插嘴，請優先調高 **`EARLY_CHECK_MS`**。
> - 如果你覺得 AI 的反應太慢，講完話要等太久才回答，請調低 **`EARLY_CHECK_MS`** 或 **`MAX_STOP_MS`**。
